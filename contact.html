Vad är en front-end utvecklare?
För att förstå yrkesrollen på vad front-end developer innebär har jag valt att undersöka och analysera företaget Wasabi Web. Fokusen kommer ligga kring företaget, men jag har även jämfört företaget med andra källor för att få bättre bild på det hela. Front end utvecklare kodar sidan som användare integrerar med genom programmerings språk så som html, css och javascript. Sedan finns det även backend utvecklare som jobbar lite mer med det som händer bakom sidan och web designers som man ofta blandar ihop med front end utvecklare. Det finns web designers som också kodar men dem är lutade åt visual/UI eller user interface design, alltså dem som designar utseendet på websidan. Long story short, så är back end utvecklare dem som får allt att fungera på bakgrunden av sidan (PHP, SQL etc), web designers är dem som designar det visuella på websidan och front end utvecklare är dem som designar websidan och med andra ord kodar så att allt sitter på plats och all integration med användare på web sidan.

Detta är då lite kort om dessa roller som finns inom webben. Det finns företag som söker front end utvecklare och oftast nämner vilka tekniker eller kunskaper som krävs. Dock kan kunskaperna som vissa företag kräver vara mer än av vad en front end utvecklare innebär, so som t.ex företaget Wasabi Web som jag undersökt. Om man kollar på vad Wasabi Web (http://wasabiweb.se/webbyran-wasabi-web-soker-front-end-utvecklare/) kräver för att söka till deras front end utvecklar tjänst, så kräver dem förutom de basic, html, css, jacascript och jquery, även photoshop, illustrator och även back end språk som PHP, MySql och WordPress. Detta får en att undra om dem söker en front end utvecklare eller en full stack utvecklare som kan båda front och back end. Visserligen går det mycket smidigare om en front end har en hum om hur allt annat angående utvecklingen av webben eller företagets tjänst fungerar eftersom man som front end utvecklare eller utvecklare överhuvudtaget för det mesta jobbar i team där front end utvecklar har sin roll, och det är då som sagt design på sidan genom kodning. Tillsammans med andra utvecklare i teamet och genom alla deras roller och arbete skapar/bildar olika projekt, produkter eller tjänster. Det kanske är det Wasabi Web är ute efter när dem sätter dem kraven, men då skulle dem ha fokuserad mer på front end kraven och satt "förståelse" av dem andra delarna som ett mindre krav, för annars blir det som sagt en full stacked utvecklare dem söker och inte en front end.

Front end utvecklare är utvecklare som utvecklare fronten på en hemsida, men det är inte förrän under det senaste dekaden som titeln "front end utvecklare" kommit till liv. Förr fanns det inte direkt distinktion mellan front end och back end. Man skulle nog kunna säga att det var mer "full stacked" stil då, men vid den fanns det inte lika mycket man behövde kunna så som idag så att en direkt distinktion mellan front end och back end behövdes, utan en kille räckte för att skapa sidan och fokusen på design var inte lika stor. Anledning till det är att förr först och främst var teknologin inte vad den är idag. Majoriteten av folket använde endast internet explorer till skillnade från och idag då vi har flera val webb läsare så som chrome, firefox, safari osv. Dessutom fanns det inga smartphones eller tablet, mindre devices helt enkelt. Språken man använde var inte lika avancerade och många utan webben var för det mesta html och css. Bara användning och användnings metoder av webben var mycket mindre då. Går man fram i tiden lite grann så började olika nya webläsare att skapas som kunde konkurera med internet explorer. I mitt minne är det första jag minns mozilla firefox. Det var mycket snabbare och smidigare att använda. Sedan kom även chrome osv. För användare kan det vara en god nyhet med snabbare web läsare men för utvecklare så kan det vara något att skrapa i huvudet för. Anledning till det är för att web sidor vid den tiden oftast visades på olika sätt än vad utvecklaren hade tänkt. Det beror på att dem olika web läsarna har olika så kallade "browser -rending engines". Internet explorer använder trident, firefox använder gecko, safari och google chrome använder webkit. Vad det innebär är att utvecklarna måste skapa hemsidorna så att dem kan anpassa sig efter olika web läsare. Kompabilitet problem kunde lösas genom flera stylesheets för olika web läsare eller specialiserade css kod som fokuserar på specific web läsare. Allt detta blev dock mycket enklare med upkomsten av ett library i javascript så kallade jquery. Det gjorde så att man inte behövde tänka så mycket på vilka web läsare användarna som besöker sidan använder och även gjorde api management mycket enklare. Förutom upkomsten av fler web läsare och utveckling av programmerings språken så kom även vad vi alla nästan använder idag, smartphones och tablets. Att använda webläsare i smartphones eller tablets är annorlunda än att använda webläsare på vanlig desktop eller laptop. I smartphones och tablet behöver man zooma in och ut ganska ofta och tillskilnade från vanliga datorer använder man int sig av mus utan man drar och pekar på skärmen vilket är ännu en grej att tänka på när man skapar web sidor. Även olika tänkanden som "mobile first" när man skapar web sidorna, för funkar den på smart phones, så funkar den på desktop. Dessutom responsiv design. Tillskillnad från hur det var då och hur det är nu så finns det betydligt många mer företag som använder sig av webben för att marknadsföra, sälja produkter, sälja tjänster med mera. Nu finns det nästan inget företag, iallafall dem större men även mindre företag, har en websida där dem antingen marknadsför deras företag eller även säljer deras produkter förutom på butiken. Detta har gjort så att web marknaden blivit betydligt mycket större. Någonstans emellanåt här av all det som har nämnts har distinktionen mellan front end och back end utveckling blivit mycket tydligare, eller så att säga dem har blivit 2 helt olika roller. Med utvecklings gång har det blivit mer fokus på fronten av websidan som b.la design, interaktionen etc. För en person att hålla reda på allt dessa är kanske inte omöjligt om man är tillräckligt bra och kan båda delarna hyfsat men det är väldigt mycket man måste kunna och för att få varje del att bli så bra som möjligt så har man separerat på front end och back end.

Wasabi Webs tjänster fokuserar på att hjälpa företag att öka deras försäljning med hjälp av b.la front end utveckling. Att jobba som front end utvecklare i en web byrå så som Wasabi Web så blir det nog för det mesta utveckling av web sidor för andra företag som vill synas i webben och marknadsföra. Vår jobb som front end utvecklare blir då att skapa dessa sidor och få front delen att se bra ut som möjligt och tillsamans med utvecklings teamet göra tjänsten möjlig.
